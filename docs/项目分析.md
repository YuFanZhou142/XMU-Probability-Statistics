# 项目分析

## 1. 项目定位

该仓库对应厦门大学通信工程专业《概率统计》课程的期末课程设计，主题是“随机变量的计算机产生”。项目核心目标是利用 MATLAB 完成随机变量模拟、分布可视化和实验分析。

从原始提交内容来看，这不是一个“长期维护型软件项目”，而是一份典型的课程作业提交包。它更偏向于“实验材料集合”，而不是“可复现、可阅读、可扩展”的标准仓库。

## 2. 原始仓库包含的内容

原始仓库只有 4 个顶层文件：

1. `README.md`
2. `code.rar`
3. `自学内容安排.docx`
4. `课程设计报告.docx`

进一步展开后，可以确认项目实质上包含以下三类内容：

1. 课程要求文档
   `自学内容安排.docx` 说明了作业背景、任务范围、提交方式和截止时间。
2. 实验报告文档
   `课程设计报告.docx` 记录了实验背景、四个任务的原理、代码、结果与结论。
3. MATLAB 实验脚本
   原压缩包中的代码包含 6 个 `.m` 文件，覆盖 4 个实验任务。

## 3. 实验内容拆解

根据作业要求和实验报告，项目主要完成了以下四个实验主题：

### 3.1 任务一：Wichmann-Hill 均匀分布随机数生成

目标是通过三重同余的 Wichmann-Hill 算法生成 `(0,1)` 上的伪随机数，并观察均匀性与独立性。

原始代码中：

1. `T_1.m` 用两组种子生成均匀随机数并绘制直方图。
2. `T_1_analysis.m` 在 `T_1.m` 基础上补充了散点图、均值和方差统计。

该部分体现出项目对“伪随机数生成机制”的理解，是后续所有分布构造的基础。

### 3.2 任务二：Rayleigh 分布随机变量生成

目标是用逆变换法将均匀分布随机变量转换为 Rayleigh 分布随机变量，并观察不同 `sigma` 下的分布变化。

原始脚本 `T_2.m` 完成了样本生成和直方图绘制，但没有叠加理论曲线，也没有输出数值统计结果。

### 3.3 任务三：高斯分布随机变量生成

目标是使用 Box-Muller 方法构造高斯分布样本，并比较样本直方图与理论正态分布。

原始代码中：

1. `T_3.m` 实现了基础版 Box-Muller 采样和样本方差输出。
2. `T_3_ananlysis.m` 在此基础上叠加理论正态曲线。

这一部分已经具备一定的“实验分析”意识，但命名存在拼写错误，且依赖 `normpdf`，在没有统计工具箱的环境下可能无法运行。

### 3.4 任务四：离散随机变量生成

目标是根据给定分布律，通过累积分布函数映射生成离散随机变量样本，并验证实验频率是否接近理论概率。

原始脚本 `T_4.m` 通过累积分布逐点查找区间，能够完成样本生成和直方图绘制，思路是正确的。

## 4. 原始项目的主要问题

从“课程作业包”转为“标准 GitHub 项目”时，原仓库主要存在以下问题：

1. 目录过于扁平
   所有内容都堆在根目录，不利于区分文档、源码、归档文件和结果产物。

2. 存在重复与冗余
   `code.rar` 与实际脚本内容重复，文档也只有二进制格式，GitHub 浏览体验较差。

3. 命名不统一
   代码文件采用 `T_1`、`T_2` 这类作业式命名，含义不直观；`T_3_ananlysis.m` 还存在拼写错误。

4. 缺少统一入口
   用户需要逐个打开脚本执行，项目没有“统一运行全部实验”的入口文件。

5. 可复现性不足
   除任务一外，其余脚本没有固定随机种子；不同机器运行后得到的图形和统计结果会有差异。

6. 可移植性不足
   任务三直接调用 `normpdf`，在部分 MATLAB 环境下可能因缺少 Statistics Toolbox 而报错。

7. README 过于简略
   原始 `README` 只有一句简介，没有背景、目录、运行方式、实验说明和结果组织。

8. 缺少结果目录与输出约定
   原始脚本只画图不落盘，难以在仓库中保留可查看的运行结果。

## 5. 标准化整理目标

本次整理的目标不是“重写课程内容”，而是在保留原始作业材料的前提下，把它整理成一个更像正式项目的仓库：

1. 保留原始提交材料，避免丢失课程作业原貌。
2. 将源码、文档、归档文件和结果目录解耦。
3. 将 MATLAB 逻辑拆分为可复用函数和清晰的任务入口。
4. 增加统一运行入口，降低复现门槛。
5. 输出图片到固定目录，使结果可以被追踪和展示。
6. 用中文重写 README，让项目在 GitHub 上可直接阅读。

## 6. 整理后的结构说明

整理后的项目将采用以下结构：

```text
.
├─ archive/
│  └─ original_submission/      原始作业归档
├─ docs/
│  ├─ original_submission/      原始文档
│  └─ 项目分析.md               当前分析说明
├─ results/
│  └─ figures/                  MATLAB 运行后生成的图片
├─ scripts/                     各任务入口与总入口
├─ src/                         可复用 MATLAB 函数
├─ .gitignore
└─ README.md
```

## 7. 整理后项目的价值

整理之后，这个仓库将具备以下明显提升：

1. 对课程老师或同学更友好
   打开仓库即可看到任务背景、目录说明和运行方式，不需要先下载压缩包。

2. 对你自己更友好
   未来你再回顾这份课程设计时，可以快速定位“原始材料”“整理后的代码”“生成结果”。

3. 对后续扩展更友好
   如果你以后想继续补实验、换 Python 实现、增加结果分析，新的结构可以直接承接。
